## Phase 1: Headless Rust Packet Parser

**Goal**: A standalone Rust library that parses raw IP packets and extracts domains. Fully testable on macOS with `cargo test`.

**Specs to read**:
- `local-packet-engine.md` — Full Rust implementation for all modules
- `tech-stack.md` — Rust version, Cargo dependencies, crate choices

### Step 1.1: Scaffold the Rust Project

**Spec**: `tech-stack.md` (Rust Dependencies section), `project-structure.md` (rust/ directory)

**Create**:
```
rust/packet_engine/
├── Cargo.toml          # Copy from tech-stack.md "Cargo.toml" section
├── Cargo.lock          # Generated by cargo
├── build.rs            # Placeholder (cbindgen comes in Phase 2)
├── cbindgen.toml       # Placeholder
└── src/
    ├── lib.rs          # Module declarations only (pub mod engine; etc.)
    ├── engine.rs       # Empty struct placeholder
    ├── ip.rs           # Empty
    ├── dns.rs          # Empty
    ├── tls.rs          # Empty
    ├── tcp_reassembly.rs # Empty
    ├── domain.rs       # Empty
    ├── storage.rs      # Empty
    ├── errors.rs       # Empty
    └── constants.rs    # Empty
```

**Acceptance**: `cargo build` succeeds. `cargo test` runs (0 tests).

### Step 1.2: Implement Constants and Error Types

**Spec**: `local-packet-engine.md` — "Constants" and "Error Types" sections

**Files**: `src/constants.rs`, `src/errors.rs`

**Acceptance**: All constants defined. `EngineError` enum with `thiserror` derives compiles.

### Step 1.3: Implement Domain Record Model

**Spec**: `local-packet-engine.md` — "Domain Record" section

**Files**: `src/domain.rs`

**Implement**:
- `DomainRecord` struct (domain, timestamp_ms, source)
- `DetectionSource` enum (Dns, Sni)
- `DomainRecord::from_raw_name()` — normalization, validation, noise filtering
- `is_noise_domain()` — system noise filter

**Acceptance**: Unit tests pass:
- `"google.com."` → `"google.com"` (trailing dot stripped)
- `"AMAZON.COM"` → `"amazon.com"` (lowercased)
- `"apple.com"` → `None` (noise filtered)
- `"192.168.1.1"` → `None` (IP literal rejected)
- `"localhost"` → `None` (no dot, rejected)

### Step 1.4: Implement IP Header Parser

**Spec**: `local-packet-engine.md` — "IP Header Parser" section

**Files**: `src/ip.rs`

**Implement**:
- `IpHeader` struct
- `IpProtocol` enum (Tcp, Udp, Other)
- `parse_ip_header()` — handles IPv4 and IPv6
- `parse_ipv4()`, `parse_ipv6()` internal functions

**Acceptance**: Unit tests with raw packet bytes:
- IPv4 TCP packet → correct src/dst IP, protocol=Tcp, correct payload_offset (20+)
- IPv4 with options → correct variable header length
- IPv6 packet → correct 40-byte header, correct next_header
- Truncated packet → `None`

### Step 1.5: Implement DNS Query Parser

**Spec**: `local-packet-engine.md` — "DNS Query Parser" section

**Files**: `src/dns.rs`

**Implement**:
- `parse_dns_query()` — extracts domain names from DNS Question section
- `parse_dns_name()` — wire format name parsing with compression pointer support

**Acceptance**: Unit tests with captured DNS packets:
- Standard A query for "google.com" → `vec!["google.com"]`
- AAAA query for "ipv6.example.com" → `vec!["ipv6.example.com"]`
- DNS response (QR=1) → empty vec (we only parse queries here)
- Compressed name → correctly resolved
- Truncated packet → empty vec (no panic)

### Step 1.6: Implement TLS SNI Extractor

**Spec**: `local-packet-engine.md` — "TLS SNI Extractor" section

**Files**: `src/tls.rs`

**Implement**:
- `extract_sni()` — walks TLS ClientHello extensions to find SNI (type 0x0000)
- `parse_sni_extension()` — parses the SNI extension payload
- `is_tls_client_hello_start()` — quick check for TCP reassembly

**Acceptance**: Unit tests:
- TLS 1.2 ClientHello with SNI "amazon.com" → `Some(DomainRecord("amazon.com"))`
- TLS 1.3 ClientHello (record layer says 0x0301) → SNI extracted correctly
- ClientHello without SNI extension → `None`
- Non-handshake TLS record → `None`
- Truncated data → `None` (no panic)

### Step 1.7: Implement TCP Reassembly

**Spec**: `local-packet-engine.md` — "TCP Reassembly" section

**Files**: `src/tcp_reassembly.rs`

**Implement**:
- `FlowKey` struct (src_ip, dst_ip, src_port, dst_port)
- `FlowBuffer` internal struct
- `TcpReassembler` with `process_segment()`, LRU eviction, timeout eviction
- `ReassemblyResult` enum (Complete, NeedMore, NotRelevant)

**Acceptance**: Unit tests:
- Single-segment ClientHello → `Complete(DomainRecord)` immediately
- Two-segment ClientHello → `NeedMore` then `Complete`
- Non-TLS payload → `NotRelevant`
- Fill to `MAX_TCP_FLOWS` → oldest flow evicted
- Flow older than `FLOW_TIMEOUT_SECS` → evicted
- Buffer exceeds `MAX_FLOW_BUFFER_BYTES` → flow dropped

### Step 1.8: Implement SQLite Storage

**Spec**: `data-storage.md` — "Rust Storage Module" section, full schema

**Files**: `src/storage.rs`

**Implement**:
- `DomainStorage::new(db_path)` — open DB, set pragmas, create tables
- `DomainStorage::batch_insert()` — UPSERT domains + insert visits
- `DomainStorage::domain_count()`
- `DomainStorage::cleanup_old_visits()`

**Acceptance**:
- Create in-memory DB → tables exist
- Insert 3 records → `domain_count() == 3`
- Insert same domain again → `domain_count()` still 3, `visit_count` incremented
- `cleanup_old_visits()` removes old entries

### Step 1.9: Implement Engine Orchestrator

**Spec**: `local-packet-engine.md` — "Engine Core" section

**Files**: `src/engine.rs`

**Implement**:
- `PacketEngine::new(db_path)` — initializes all subsystems
- `PacketEngine::process_packet(packet)` — full pipeline
- `handle_udp()` — DNS query detection and parsing
- `handle_tcp()` — TLS SNI extraction with reassembly
- `maybe_flush()` / `flush()` — batch write to SQLite
- `EngineStats` struct

**Acceptance**: Integration test:
- Create engine with in-memory SQLite
- Feed it a DNS query packet → domain in DB
- Feed it a TLS ClientHello packet → domain in DB
- Verify stats counters are correct

### Step 1.10: Create Test Fixtures and Integration Tests

### Step 2.2: Implement FFI Entry Points

**Spec**: `cross-platform-bindings.md` — "Rust FFI Functions" section (full `lib.rs`)

**Files**: `rust/packet_engine/src/lib.rs`

**Implement all `extern "C"` functions**:
- `packet_engine_init(db_path)` → `*mut PacketEngine`
- `packet_engine_destroy(engine)`
- `packet_engine_process(engine, data, len, out_buf, out_cap, out_len)` → `c_int`
- `packet_engine_process_batch(...)` → `c_int`
- `packet_engine_flush(engine)` → `c_int`
- `packet_engine_set_ech_downgrade(engine, enabled)` → `c_int`
- `packet_engine_get_stats(engine)` → `EngineStatsFFI`
- `packet_engine_last_error()` → `*mut c_char`
- `packet_engine_free_string(s)`

**Key requirement**: Every function wraps its body in `panic::catch_unwind`. No panic may cross the FFI boundary.

**Acceptance**: `cargo build` succeeds. Generated header matches spec in `cross-platform-bindings.md`.

```bash
./scripts/build-rust.sh aarch64-apple-ios-sim
```

**Acceptance**: Both `.a` files produced without errors.

### Step 2.4: Create Xcode Project

**Acceptance**: `xcodebuild` compiles both targets without linker errors.

**Spec**: `ech-fallback.md` — "Layer 0" and "dns_filter.rs" sections

**Files**: `rust/packet_engine/src/dns_filter.rs`

**Implement**:
- `filter_ech_from_dns_response()` → `FilterResult`
- `quick_scan_for_https_rr()` — fast path for common (non-HTTPS RR) responses
